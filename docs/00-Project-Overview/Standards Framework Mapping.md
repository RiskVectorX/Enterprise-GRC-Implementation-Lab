# Executive Summary

This document defines the standards, frameworks, and regulatory requirements adopted throughout the Enterprise GRC Implementation Lab.

Rather than relying on a single framework, this engagement combines internationally recognized governance, risk management, security, and privacy standards to create a comprehensive enterprise GRC program. Each framework serves a distinct purpose and is applied where it provides the greatest value.

---

# Purpose

The objectives of this document are to:

- Identify the frameworks used throughout the engagement.
- Define the purpose of each framework.
- Explain how frameworks complement one another.
- Map project deliverables to their supporting standards.
- Demonstrate alignment with recognized industry practices.

# Framework Roles

## NIST RMF

Provides the overall lifecycle for planning, implementing, assessing, authorizing, and continuously monitoring security controls throughout the engagement.

Primary application:

- Project lifecycle
- Governance structure
- Continuous monitoring

---

## NIST SP 800-30 Rev.1

Provides the methodology for identifying threats, vulnerabilities, likelihood, impact, and enterprise risks.

Primary application:

- Risk Assessment Methodology
- Enterprise Risk Register
- Risk Treatment Planning

---

## NIST SP 800-53

Provides a catalog of security and privacy controls that support risk treatment decisions.

Primary application:

- Control Selection
- Security Baselines
- Control Assessment

---

## ISO/IEC 27001:2022

Defines the requirements for establishing, implementing, maintaining, and continually improving an Information Security Management System (ISMS).

Primary application:

- ISMS Scope
- Governance
- Policies
- Statement of Applicability
- Internal Audit

---

## ISO/IEC 27002:2022

Provides implementation guidance for information security controls selected during the engagement.

Primary application:

- Security Policies
- Operational Controls
- Best Practices

---

## PCI DSS

Applies to systems and processes involved in payment card processing.

Primary application:

- Payment Security
- Access Control
- Logging
- Encryption

---

## GDPR

Supports privacy governance for personal data belonging to individuals located within the European Union.

Primary application:

- Privacy Controls
- Data Subject Rights
- Data Protection Principles

---

## Digital Personal Data Protection (DPDP) Act

Supports compliance with India's Digital Personal Data Protection Act.

Primary application:

- Privacy Governance
- Consent Management
- Personal Data Protection
- Data Processing Controls

# Framework Integration

The engagement does not treat these standards independently.

Instead:

- NIST RMF provides the project lifecycle.
- NIST SP 800-30 provides the risk assessment methodology.
- NIST SP 800-53 provides security controls.
- ISO/IEC 27001 establishes the management system.
- ISO/IEC 27002 provides implementation guidance.
- PCI DSS addresses payment security.
- GDPR and the DPDP Act address privacy and data protection obligations.

Together, these frameworks establish a comprehensive Governance, Risk & Compliance program.
